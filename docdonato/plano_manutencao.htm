Procedimentos de Manutenção

 

1. Introdução

 

Este documento tem como função fazer uma breve explanação do ambiente da aplicação Donato 3.0, de forma a dotar os técnicos que irão fazer sua manutenção de informações dobre sua estrutura.

 

É importante também ter acesso aos demais documentos que compõem a documentação técnica do Donato 3.0, em particular do modelo E/R do banco de dados.

 

Estes arquivos pdem ser encontrados junto com os documentos de planejamento e desenvolvimento do Donato no diretório C:\Program Files\Apache Group\Apache\htdos\donato\docdonato.

 

2. Ambiente

 

O Donato 3.0 foi totalmente desenvolvido em PHP e os dados armazenados em um banco de dados MySQL. Como servidor web foi utilizado o Apache. As versões utilizadas podem ser consultadas no documento “Componentes e Requisitos” da documentação técnica.

 

O modelo de dados utilizado no Donato 3.0 também está disponível na documentação técnica, assim como as definições de chaves primárias e relacionamentos.

 

Uma vez entendido os pré-requisitos de ambiente e o modelo de dados, pasaremos a aplicação propriamente dita.

 

3. A Aplicação

 

O Donato 3.0, como dito acima, é uma aplicação desenvolvida em PHP. Para sua execução é necessário que seja acessada por um navegador (browser) compatível (ver lista no documento “ Componentes e Requisitos”).

 

Os arquivos que contém as páginas do projeto devem estar no diretório de documentos do Apache, no sub-diretório donato.  Existe uma árvore de diretório que deve ser obedecida.

O diretório classes contem os arquivos de classes e funções que são utilizadas pela aplicação. Estes arquivos podem ser chamados de diversas outras páginas através de includes. Um exemplo é o arquivo classe_padrão.php, que contem a classe de conexão ao banco MySQL:

 

<?

 class conexao

 {

        {

 

         $this->link = mysql_connect($this->host,$this->user,$this->pass) or die ("Configuracao de Banco de Dados Errada!");

         mysql_select_db($this->db,$this->link);

        }

 

        function query($sql){

 

            $this->result = mysql_query($sql);

           

///////////////

           $this->Row   = 0;

           $this->Errno = mysql_errno();

           $this->Error = mysql_error();

           if (!$this->result) {

                           $this->halt();

                                               }

           return $this->result;

        }

 

function halt() {

    $this->Error = @mysql_error($this->link);

Os arquivos com classes e funções são extremamente úteis no desenvolvimento de sistemas pois permitem o reaproveitamento de código de uma forma extremamente simples. Através de uma chamada de include o arquivo é “carregado” para dentro da página chamadora. Assim, basta alterar o arquivo que contenha a função para que a mesma seja alterada em todos os pontos onde é utilizada no sistema.

O diretório imagens e seus sub-diretórios contém as imagens das obras, autores e demais arquivos de imagens que são utilizadas pelo sistema.

O diretório js contém arquivos com códigos javascript que são utilizados pela aplicação. Javascript são ódigos que são executados no browser, sem a necessidade de ida ao servidor para sua execução. Um exemplo são as validações de campos de formulários. Por exemplo: saber se uma data é válida. Neste diretório temos os seguintes arquivos:

A linhas entre os caracteres <? e ?> são os códigos PHP. Note que ao final das linhas PHP existe um caracter ; (ponto-e-vírgula).

 

A primeira linha de código PHP mostrada no exemplo é exatamente a carga do arquivo classe_padrão.php. A partir deste momento, as referências as classes e funções presentes no arquivo de classes podem ser feitas como se a mesma estivesse no arquivo index.php.
