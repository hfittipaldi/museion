Manual do Administrador

 

1. Introdução

 

Este documento tem como finalidade dar material ao administrador do Sistema Donato quando a criação de usuários, manutenção de menus e procedimentos de segurança (backup).

 

2. Usuários no Donato

 

O sistema Donato é dotado de um controle de usuário bastante sofisticado. É possível definir o nível de acesso aos menus do sistema bem como definir as coleções que um dado usuário terá acesso.

 

2.1. Logando no Sistema

 

Para identificar o usuário o sistema solicita na tela de abertura a sua identificação. Esta identificação é feita a partir de um login e senha.

 

fig. 1 – Login do sistema

 

Uma vez identificado o usuário tem acesso ao sistema com o menu sendo apresentado de acordo com as definições dadas para o usuário logado.

 

fig. 2 – Usuario com permissão de acesso restrita. No exemplo o usuário VISITANTE.

 

Uma vez logado no sistema, pode-se fazer o logout (saída do sistema) clicando-se no ícone que está ao lado do nome do usuário logado.

 

2.2. Administrando Usuários

 

A criação de usuários no Sistema Donato é feita a partir do login de um usuário administrador. Uma vez logado no sistema, o usuário terá acesso ao menu Estrutura. Uma vez posicionado sobre este item de menu serão apresentadas as opções de Usuáriosm Alteração de Senha, Menus e Parâmetros conforme a tela abaixo:

 

fig. 3 – Menu de Estrutura de Sistema do Donato

 

2.2.1. Criando um novo Usuário

 

Para criar um novo usuário devemos entrar na opção Usuário do Menu Estrutura/Sistema. Uma vez selecionada esta opção veremos a tela com os usuários já criados e, a partir desta tela podemos excluir o usuário, alterar seus dados cadastrais ou alterar suas permissões de acesso. Para criar um novo usuário devemos clicar no ícone de adição de usuários (fig. 4).

 

fig. 4 – Tela de Usuários do Sistema Donato

 

Veremos então a tela para o cadastramento dos dados básicos do Usuário. Preencha a tela com as informações solicitadas:

 

fig. 5 – Tela de Dados Básicos do Usuário

 

Os campos solicitados nesta tela são os seguintes:

 

Campo

Definição

Nome

Nome completo do Usuário

Nascimento

Data de Nascimento do Usuário

Email

Endereço de e-mail do Usuário

Login

Código de Login para este Usuário

Senha

Senha de acesso do Usuário

Ativo

Informa se este login está ativo ou não

Status

Usuário comum ou Administrador do Sistema

 

Após preencher os campos clique no botão Gravar. Será apresentada então a próxima tela onde devemos informar os procedimentos para gerar as permissões deste usuário:

 

fig. 6 – tela de Procedimento de Permissão

 

Nesta tela devemos informar como daremos as permissões ao usuário. Temos aqui 3 opções a saber:

 

Opção

Funcionalidade

Administrar permissões de menu

Esta opção nos permite criar as permissões do usuário informando quais os itens de menu que o mesmo terá acesso.

Administrar coleções

Esta opção monta as regras de acesso as coleções para este usuário.

Copiar permissões com base nas permissões de um usuário existente

Esta opção permite que as permissões já definidas de um usuário possa ser atribuida a um outro usuário

 

 

Com a primeira opção montamos a árvore de permissão do usuário informando para cada item de menu qual o nível de acesso do usuário. Veja a tela a seguir:

 

fig. 7 – Atribuindo aceso aos menus do sistema

 

Para cada item de menu selecionado poderemos dar acesso aos sub-itens existentes. Veja na tela a seguir:

 

fig. 8 – Acesso aos sub-itens de um menu

 

Após montar as permissões neste item de menu, clique no botão Enviar no fim da página e o sistema voltará a solicitar qual menu se deseja configurar. Faça a configuração nos itens de menu que se deseja disponibilizar para o usuário.

 

 

Com a segunda opção informamos a quais coleções o usuário tem acesso para manipulação de informação:

 

fig. 9 – Configurando Permissão de Coleções

 

Na tela a seguir informe as coleções que o usuário terá acesso. Lembre de manter a tecla CONTROL pressionada para selecionar mais de uma coleção da lista:

 

fig. 10 – Informando as coleções permitidas

 

Na hipótese de um usuário não ter acesso a uma determinada coleção e tentar abrir o registro de uma obra desta coleção, será informada a restrição de acesso e o usuário terá a opção de consultar a obra:

 

Se optarmos por abrir no módulo de consulta, veremos os dados básicos da obra e sua imagem para exibição:

 

fig. 11 – Abertura da obra no modo consulta

 

Através da terceira opção podemos pegar as definições de um usuário previamente criado no sistema e replicar suas permissões para o novo usuário. Isto é especialmente útil na criação de logins com permissões semelhantes. Podemos definir perfis criando usuários com as permissões que desejamos e depois simplesmente atribuir este conjunto de permissões a outro novo usuário.

 

Selecione a opção correspondente à cópia de permissões e clique em avançar:

 

fig. 12 – Copiando permissões de outro usuário

 

Na tela a seguir informe o usuário que irá ceder suas permissões. Marque a caixa ao lado do botão Copiar Permissões para habilitá-lo (fig. 13). A seguir clique no botão que será ativado:

 

fig. 13 – Informando o usuário e copiando suas permissões

 

2.2.2. Criando um novo Administrador

 

Para criar um novo administrador no sistema devemos fazer login como usuário administrado já previamente cadastrado e entrar no menu Estrutura/Sistema/Usuário. Clicar no ícone de adição de usuário e preencher os dados do usuário. Informar na caixa de seleção Status a opção “Administrador”.

 

fig. 14 – Criando um usuário Administrador

 

Ao se optar pelo status de administrador, as permissões completas a toda a árvore de menu do sistema e a todas as coleções serão dadas automáticamente.

 

2.2.3. Incluindo novas coleções

 

Ao se incluir uma nova coleção devemos lembrar de dar acesso aos usuários que deverão manipular as obras desta coleção. Assim, deve-se entrar nas definições de cada usuário envolvido e incluir o acesso a esta coleção. Após selecionar o usuário que desejamos atualizar, clicamos no ícone de manutenção de Permissões e na tela a seguir selecionamos Administrar coleções:

 

fig 15 – Administrar coleções de Usuário

 

Após será apresentada a tela com as coleções já permitidas para este usuário. Inclua a coleção desejada selecionado-a na lista de opções. Não esqueça de manter a tecla CONTROL pressionada caso mais de uma coleção for selecionada.

 

fig. 16 – Selecionado Coleções

 

É importante lembrar de atualizar também os dados dos usuários utilizados para gerar perfil (aqueles que terão seus dados de permissão copiados para os novo usuários). 

 

Quanto aos usuários administradores, estes também devem ter suas permissões revistas na hipótese de criação de novas coleções. Para isso devemos entrar nas permissões de acesso dos usuários administradores (ícone de manutenção de Permissões na tela de Usuários) e clicar no botão Atalizar Permissões, conforme a tela abaixo:

 

fig. 17 – Atualizando Permissões de Administrador

 

2.2.4. Excluindo um Usuário

 

Para excluir um usuário no sistema, devemos primeiramente localizá-lo na lista de usuários e clicar no ícone de exclusão do usuário:

 

fig. 18 – Excluindo um Usuário

 

Após clicar no ícone de exclusão será solicitada a confirmação da exclusão do usuário. Clique no botão OK para confirmar a exclusão.

 

fig. 19 – Confirmando a Exclusão do Usuário

5. Backup e Restauração

 

Através do MySQL Administrator iremos fazer os backups (cópias de segurança) e eventuais restaurações em caso de necessidade. O MySQL Administrator tem as funções de cópia e restauração, permitindo inclusive que as cópias seja agendadas (schedule) para determinados dias e horários.

 

5.1. Cópia de Segurança

 

Entre no MySQL Administrator e acesse o item Backup:

 

 

Clique no botão New Project para abrir um novo projeto de backup. Dê um nome para este Projeto. Marque na coluna de Esquemas o banco Donato e clique no botão de cópia do banco para a área Data Directory:

 

 

Clique na aba Schedule. Marque que você deseja agendar este projeto (Schedule this backup Project). Informe o diretório onde o arquivo de backup será gerado (Target Folder) Informe a periodicidade do backup:

 

Execution Time

Periodicidade

Execute backup daily

Diário

Execute backup weekly

Semanal

Execute backup monthly

Mensal

 

Se a periodicidade for semanal, informe os dias da semana que você deseja que o backup seja executado. Se for mensal, informe o dia do mês em que você deseja que o backup seja executado.

 

Por fim informe a hora em que você deseja que o backup seja realizado. O idela é que o backup seja feito em um horário em que os usuários não estejam utilizando o sistema.

 

 

 

Clique no botão Save Project e pronto.

Nota: as imagens existentes no Donato não são armazenadas junto com o backup descrito acima. É altamente recomendado que se faça cópia das imagens existentes em discos externos (CD ou DVD).
