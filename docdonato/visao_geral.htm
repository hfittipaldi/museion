Visão Geral do Produto

 

O MNBA utilizando o profundo conhecimento no tocante ao armazenamento, manutenção e catalogação das obras em seu poder e sentindo a necessidade de fazer consultas e cruzamentos de informações sobre estas obras, desenvolveu, com uma equipe interna, um produto para esta finalidade denominado “Donato”. O Donato é um sistema escrito em Microsoft Access, tendo uma interface GUI (Graphic User Inteface) criada com as funcionalidades do próprio Access.

 

Como qualquer outro produto, na medida de sua utilização foi sendo necessária a implementação de melhorias e adições, que levaram o MNBA a procurar uma empresa que pudesse se ocupar deste trabalho de desenvolvimento, tendo como ponto de partida a concepção original do Donato e também a necessidade de ampliação do porte da base de dados para o novo ambiente.

 

Assim, o Projeto Donato 3.0 foi concebido visando à criação de um sistema cuja função será a catalogação pormenorizada do acervo de obras de uma dada instituição, incluindo aqui dados sobre os autores, banco de imagens e condições da peça. Adicionalmente, o sistema ainda fará controles como entrada e saída de obras para exposições, manutenção, restauração e recuperação de peças, tendo como condição o aproveitamento da base de dados pré existente na versão anterior do Donato e a ampliação de  suas capacidades.

 

A relação completa das funções do sistema pode ser consultada no documento “Resumo das Funções do Sistema”.

 

Como ponto de partida, o MNBA objetivava um sistema com sua operação em um navegador (browser). Desta forma, seria possível fazer uso de qualquer microcomputador disponível na rede sem a necessidade de instalação de softwares adicionais e também este seria o caminho para futuramente disponibilizar o acesso através da internet a este mesmo sistema.

 

 A completa abordagem quanto à interface utilizada no projeto pode ser consultada no documento “Especificação da Interface”, que trata não somente da apresentação, mas também de características técnicas exigidas, como resolução em pixels da interface de vídeo, profundidade de cores, tamanho das áreas de interface e operação do sistema, logos e imagens de fundo.

 

Visto a complexidade da interface pretendida pelo MNBA, O Donato 3.0 demanda   um navegador com os recursos necessários à sua correta apresentação. Devido à não padronização de diversas funções que são necessárias para a construção da interface proposta pelo MNBA, ficaram as opções de navegador restritas ao Microsoft Internet Explorer e Mozzila Firefox nas versões especificadas no documento “Componentes e Requisitos”.

 

As ferramentas escolhidas para o desenvolvimento do sistema são em sua maioria de código aberto e aderentes ao conceito de software livre. Todos os requisitos de software para a operação do sistema estão especificados no documento “Componentes e Requisitos”, assim como também estão definidos no mesmo documento os requisitos de hardware.

 

A partir de entrevistas com os principais usuários e também com o desenvolvedor do projeto Donato original, especificou-se todo o escopo de operação do sistema, gerando para tanto o modelo entidade/relacionamento conforme o documento “Donato_3_Rel_Completo”. Adicionalmente os documentos de Chave Primária, Propriedades e Relacionamentos do modelo esclarecem a questão do banco de dados especificado.
