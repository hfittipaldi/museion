Componentes e Requisitos

 

 

1.     Introdução

 

A especificação do Donato 3.0  é baseada na versão em uso atualmente no MNBA, desenvolvido em MS-ACCESS (banco e apresentação).

 

É importante notar que devido à tecnologia adotada para o desenvolvimento e as novas especificações feitas que as versões 2.0 e 3.0 não serão exatamente idênticas. Deverão ser guardadas as devidas diferenças entre elas, inclusive por conta da interface de uso da aplicação.

 

A partir desta versão foi criada a estrutura para a versão 3.0, que contemplará todas as facilidades atualmente existentes no produto, acrescida das facilidades definidas no documento de Resumo das Funções do Sistema.

 

2.     Projeto

 

O projeto tem como pré-requisito o uso através de browsers e a ampla utilização de software livre. Para tanto se elegeu o conjunto Browser-Apache-PHP-MySQL para o desenvolvimento do produto, seguindo as definições indicadas pela consultoria do projeto. Para melhor entendimento, abaixo segue a função de cada produto no projeto:

 

Produto

Função

Apresentação e validações

www.microsoft.com e www.mozzila.org

Devido a não-regulamentação de certas funções nos browsers e ainda devido ao layout definido para o projeto, foram eleitos os browsers Internet Explorer e Mozzila Firefox como os que serão suportados na aplicação.

 

O Apache como Web Server é utilizado atualmente em 60% dos sites publicados na Web, e ainda é reconhecido pela sua  segurança quanto a ataques e imunidade a diversos vírus que afetam outros produtos.

 

O PHP é reconhecido pela sua flexibilidade quanto a interpretação de scripts e ser um produto nativamente desenvolvido para produção de aplicações web e de grande integração com o Web Server Apache.

 

Já o MySQL é reconhecido pela sua leveza, robustez, facilidade de gerenciamento e capacidade de integração com os produtos anteriores.

 

3.     Classificação

 

Podemos classificar o produto Donato 3.0 como sendo uma aplicação com uma camada de apresentação, uma camada de processamento e uma camada de armazenamento. A camada de apresentação roda nas estações, nos browsers. A camada de processamento roda no servidor web e a camada de armazenamento roda no servidor MySQL, sendo que os dois últimos podem estar rodando ou não no mesmo servidor físico (CPU).

 

4.     Requisitos de Hardware e Software básicos

 

Temos os seguintes requisitos mínimos para as estações de trabalho e servidores:

 

Estação de Trabalho

  • Processador Pentium Celeron ou AMD Semprom;
  • 64 MB de memória RAM;
  • Monitor de vídeo padrão VGA;
  • Placa de vídeo com suporte para resolução mínima de 800X600 pixels e 256 cores;
  • Mouse e Teclado;
  • Sistema Operacional Windows ou Linux com ambiente gráfico;
  • Browser IE versão 6.0 ou Firefox 1.5.0.1
  • Conexão de rede;

 

Servidor Apache/Mysql

  • Processador Pentium (4 ou Xeon) ou AMD (Athlon);
  • 256 MB de memória RAM não compartilhada;
  • Monitor de vídeo padrão VGA;
  • Placa de vídeo com suporte para resolução mínima de 800X600 pixels e 256 cores;
  • Mouse e Teclado;
  • Sistema Operacional Windows (2000 ou 2003) ou Linux com ambiente gráfico;
  • Conexão de rede;
  • Servidor Apache versão 1.3, PHP versão 5.0.3 e Banco de Dados MySQL versão 4.1.14;
