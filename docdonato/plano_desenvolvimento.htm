Plano de Desenvolvimento

 

1. Introdução

 

O plano de desenvolvimento do Projeto Dontato 3.0 tem por objetivo apontar diversas características do projeto, como sua interface, seu modelo de dados, a alocação de recursos, pesos do desenvolvimento, cronograma,  considerações e faormas de registros de negociações.

 

Ao longo do documento são feitas referência a outros documentos que compõem o material final desta documentação. É importante fazer o devido apontamento a estes materiais e referência-los para que este documento fique consistente.

 

2. Características do Software

 

A partir das reuniões com os usuários e a apreciação da versão do Donato 2.0, atualmente em uso no MNBA, criou-se a especificação para o desenvolvimento desta nova versão do software de registro do acervo do Museu Nacional de Belas Artes.

 

Tendo como premissa o desenvolvimento voltado para uso em um browser, definiu-se uma interface de uso da aplicação. Esta interface está demonstrada no documento “Especificação da Interface” onde podemos encontrar todas as características da mesma, incluindo as áreas de visualização, menus imagens e tamanhos.

 

A definição do banco de dados foi feita pensando no aproveitamento da base de dados atual e prevendo as necessárias alterações no software. Foram incorporados diversos mecanismos de relacionamento que não estão presentes na versão atual, visando garantir maior integridade as informações armazenadas.  Para maiores informações sobre o banco de dados favor remeter-se aos documentos de relacionamento e referências do banco de dados, assim como ao modelo E/R.

 

Visto o interesse na adoção de ferramentas livres, foram eleitos os softwares PHP para o desenvolvimento da interface e programação da aplicação e o MySQL para banco de dados. Para maiores informações sobre os softwares ver o documento “Componentes e Requisitos”.

6. Considerações quanto ao Desenvolvimento

 

O desenvolvimento do sistema uma fase crítica do projeto, pois ela deve ser fiel a especificação e atender aos anseios do usuário, levando-se em consideração o prazo do projeto e seus custos.

 

É nesta fase que tende-se a incluir ou mudar as especificações já feitas, o que pode afetar o projeto, refletindo em aumento de prazo e custo. Tais alterações, caso necessárias, devem ser alvo de negociações, de forma que o projeto comporte tal modificação e sejam ajustados seus prazos e custos.

 

 

A parte de desenvolvimento de um projeto é subdividida em diversas etapas. A definição de uma interface é apenas uma perte do necessário trabalho para se montar a interface real com o usuário, visto que facilidades de navegação, formas de apresentação, paginações e outras funcionalidades somente terão suas reais materializações aferidas quando do desenvolvimento de uma primeira tela do sistema.

 

Normalmente usam-se módulos mais simples para que estas interfaces sejam completadas a aferidas, de forma que o projeto todo se beneficiará com este trabalho inicial, e em caso de ser detectado algum problema, este pode ser contornado sem grande perda de tempo ou esforço técnico de desenvolvimento.

 

Existe uma outra fase bastante crítica no projeto Donato 3.0, que é a migração dos dados. Esta fase é crítica devido a necessidade de obediência ao valor dos dados, transportando-os para a nova estrutura e em alguns momentos transformando-os de simples campos de texto para estruturas com validação, regras e integridades.

 

Como existem diversas bases de dados, é impossível ter em um certo momento anterior ao trabalho de migração todas as bases disponíveis para uma pesquisa. Deve-se trabalhar então com um modelo de base padrão e dela ser desenvolvido todos os programas que auxiliarão na tarefa de migração.

 

7. Registros de Entrevistas e Reuniões

 

As reuniões e entrevistas com os usuários e administradores do sistema foram registradas em atas e relatórios que fazem parte da documentação do projeto, já que elas refletem as informações que foram passadas e também retratam as visões e impressões dos envolvidos.

 

Estas atas e relatórios, ao final do projeto, devem ser anexadas a documentação do mesmo, sendo assim definitivamente assimiladas pelo projeto.
